1. What is Java

  • Short answer, Java is a programming language. To be precise, it's an object oriented language. It has also some functional programming in it.
  • Objected oriented means four things:
    1. Abstraction
    2. Encapsulation
    3. Inheritance
    4. Polymorphism
  • it does not support multiple inheritance, meaning one class can only extend one other class
  • Java is a statically typed language:
    • any type of variable you create is known at compiler time. Even if you define a type of var, the compiler will guess the type of this variable.

2. What is the Object class in Java

  • the super class of every class you can create.
  • when you create a class, you can explicitly extend another class with the extend keyword.
    • even the Object class
    • by default the compiler makes you extend the super class.
    • that means every class in Java extends the Object class.

3. Can you cite methods from Object

  • the class java.lang.Object has several useful methods:
    • String toString() - returns a textual representation of an object
      • its better to Override this method for your own created classes
    • boolean equals(Object) - can compare an object with an other object
      • the default is comparing only the reference of an object
    • int hashCode() - this method is used for objects in hash based containers like HashSet<E> or HashMap<E>
      • if you Override equals(Object), it's better to do the same for hashCode()
    • Class<?> getClass() - get the Class of an Object at runtime.
      • is used in Refelction
  • there are other methods that are useful as well in programming threads:
    • wait() - causes the current thread to wait until it is awakened.
    • notify() - wakes up a single thread that is waiting.
    • notifyAll() - wakes up all threads that are waiting.
  • methods you should stay away from:
    • finalize() - Deprecated!
    • clone() - creates and returns a copy of this object.

4. Differences between String and char[]

  • String:
    • String is an object with many useful methods that char[] has not
    • String is non modifiable, once you created one.
    • String is comparable with equals()
  • char[]:
    • char[] has only methods from Object
    • char[] has a fixed size after you created it, but you can replace the elements within it.
    • to compare a char[] use Arrays.equals()
    • use Arrays.toString() to print an char[]
  • create a String from char[] by passing it as an argument: 
    char[] letters = {'h', 'e', 'l', 'l', 'o'};
String hello = new String(letters);

5. Summing an array of integers

  • old fashion way using for-each loop: 
    int[] ints = {0, 1, 2, 3, 4};
int sum = 0;
for (int element; ints) {
  sum += element;
}
  • better way using a Stream and cal sum() on it: 
    int[] ints = {0, 1, 2, 3, 4};
int sum = Arrays.stream(ints).sum();
// get average
double average = Arrays.stream(ints).average().orElseThrow();
// get minimum
int min = Arrays.stream(ints).min().orElseThrow();
// get maximum
int max = Arrays.stream(ints).max().orElseThrow();

7. How can you sort an array

  • simply call Arrays.sort(tab) 
    int[] numbersToSort = {5, 3, 6, 2, 6, 1, 9, 0};
Arrays.sort(numbersToSort);
  • sort() uses Dual-Pivot Quicksort sorting algorithms
  • don't write your own algorithm but good algorithms are:
    • QuickSort
    • HeapSort
    • TimSort
  • BubbleSort is usually a bad algorithm

8. What is the main characteristic of the String class

  • this Class is non-modifiable
    • there is no way to modify an instance of the String class once it has been created.
    • if you call toUpperCase() on a String it creates a new String.
  • to be able to modify a String use java.lang.StringBuilder
    • it is recommended that this class be used in preference to StringBuffer as it will be faster under most implementations.
    • to create a String from it call .toString() on it.
    • use + for String concatenation and .append() for StringBuilder.

9. What is the difference between == and eqauls()

  • == compares references
    • good to compare things, that are not objects:
      • int • long • char
    • dont compare float and double with ==
    • use isEqualWithinPrecision(double, double, double) 
      public static boolean isEquelWithinPrecision(final double value,
                                             final double expected, final double epsilon) {
      return (value > expected - epsilon && value < expected + epsilon);
}
    • every expected value will be seen as equals, if the difference is smaller than epsilon.
  • equals() is a method to compare Objects.

10. What about equals() and hashCode() - Relation between them

  • if two objects are equals(), they must have the same hashCode().
  • If you override equals(), then you must also override hashCode().
  • the contrary is not true, objects can have the same hashCode but are different.

11. How can you dupblicate an array?

  • you have two solutions:
    • Arrays.copyOf()
      • Copies the specified array, truncating or padding with false (if necessary) so the copy has the specified length. Calls System.arrayCopy() internally. 
            int[] array = {23, 43, 55, 12};
    int newLength = array.length;

    int[] copiedArray = Arrays.copyOf(array, newLength);
      • there is also a Arrays.copyOfRange() method. 
            int[] array = {23, 43, 55, 12};
    int newLength = array.length;

    int[] copiedArray = Arrays.copyOf(array, newLength);
    • System.arrayCopy()
      • Copies an array from the specified source array, beginning at the specified position, to the specified position of the destination array. 
            int[] array = {23, 43, 55};
    int[] copiedArray = new int[3];

    System.arraycopy(array, 0, copiedArray, 0, 3);
  • there is also a public Object clone() method you can call an some array. It returns a copy of an object, that is cast to the right type. Performance wise not efficient. 
        int[] array = {23, 43, 55, 12};
 
    int[] copiedArray = array.clone();
