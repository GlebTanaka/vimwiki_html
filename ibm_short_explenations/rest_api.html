What is a REST API

Imagine you build a web application for an ice cream shop, to show the flavors fo ice cream that are in stock that day. And allow the workers to actually make updates to those flavors. You can do this with a REST API:

  • the web app communicates with a cloud-based server via REST API.
  • REST stands for Representational State Transfer.
    • it's a standardized software architecture style, a specific type of API.
    • API (Application Programming Interface) is a way for two or more programs to communicate with each other.
    • it's all about communiation. Client with Server and vice versa.
    • RESTful web service means that a service uses REST API to communicate.
    • Benefits of REST API:
      • simple and standardized approach to communication
      • scalable ans stateless:
        • as the service gorws in complexity you can easily make modifications
        • don't need to keep track of, waht data is in wich state across client and server.
      • high performance - due to the fact that they support caching.
  • for the web app you will have an endpoint https://icecream.com/api/flavors
    • api singals, that this is a communicatin endpoint
    • flavors are recources in this REST API
  • the main building blocks of this API are:
    • request that is sent from the client to the server
      • header: to pass additional iformation about the request and client itself
      • has an operation: like GET
      • an endpoint:
      • a body or parameters: in JSON
    • response - that is rcveived back from the server typicly in JSON
    • Actions or verbs you want to make:
      • CRUD - Creat • Read • Update • Delete
        Operations SQL HTTP(REST)
        Create INSERT PUT or POST
        Read SELECT GET
        Update UPDATE PATCH or PUT
        Delete DELETE DELETE

Questions

  • how are REST APIs stateless? An appllication, API or web service that is considered stateful stores data from the client on its own server. For example if a username and password were passed from the client to the server as a from of authentication, and the server stores that data, than the web server ist statfull. This is because the server is now storing data from the client. REST architecture requires that the client state is not stored on the server. Instead each request made by the client must contain all necessary information for that particular HTTP method.
more here: https://www.spiceworks.com/tech/cloud/articles/stateful-vs-stateless/
  • Explain the HTTP methods RESTful web services will use HTTP methods and client requests. The most common methods are:
    • GET - fetches a resource from the server
    • POST - requests for a resource to be created on the server
    • PUT - requests for a resource to be updated
    • DELETE - requests for a resource to be deleted These four method correspong to CRUD opertaions
  • explain the HTTP status codes RESTful web services use HTTP status codes in server responses. The most common types of status codes are 200 codes. 200 codes represent a successful request and respons 400 codes represents a clien-side error 500 codes represents a server-side error
  • What is a URI? URI stands for Uniform Resource Identifier. It identifies every resource in the REST architecutre.
    • A URI rca be one of two types
      • URN Uniform Resource Locator - wich identifies a resource through a unique and persistent name like a books ISBN
      • URL Uniform Resource Locator- wich is your typical web address.
    • URLs are typically used when designing REST APIs
    • more here - https://de.ryte.com/wiki/URI

  • What are the diffenreces between REST and SOAP? While REST is an architecture used to develop web-services, SOAP (Simple Object Acces Protocol) serves as a protocol for exchanging structured information by way of APIs. While REST has felxible stndrads, SOAP standards are much more strict in that there implementation and statefulness often means that the client and server are closer connected. Additionally while REST allows for data transder in JSON, XML and others, SOAP only supports XML. In general, SOAPis a stricter and more Niche alternative to REST. It's used where more regulated and staple data needs to be transfered.
  • What are the differences between REST and AJAX? AJAX refers to asynchronous JavaScript and XML. Its a collectin of web technologies that allow for asynchronous web applications using the built-in XML HTTP reqeust Object. While REST API refers to an architecture for handling HTTP request, AJAX refers to a collection of web technologies for making asynchronous web requests. This means that a REST API may handle AJAX clients and that AJAX may be used to send RESTful requests. But a REST API could never be implemented nor replaced by AJAX
