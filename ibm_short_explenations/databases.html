MySQL vs MongoDB

  • Both are database management systems that let you extract data and create reports.
  • But they're also fundamentally different.
  • MySQL is a legacy table based database that has a structured System to it.
  • Structured query language (SQL) on wich the mainstay of design modeling has benn based for relational database management systems.
  • MySQL is table based and therefore it enforces a schema.
  • And the schema is used so that all rows in the table will have the same columns and those columns will have specific data types.
  • We can thing of MySQL as being a bit more rigid with its architecture.
  • Has two main components. A type of storage engine and language used to work with the data.
    • The storage engine is wehere the data is created, retrieved, sent and stored.
    • The language is how you access it.
  • MongoDB is a document based system.
  • new approach to database design representing data as JSON. So it's object based.
  • was in some ways designed to supplant the legacy MySQL structure as an easier way to work with data.
  • Felxible for formating data structures
  • is a NoSQL database. It's founded upon documentes as the unit of data for serach.
    • it epmployes JSON languag and it uses MongoDB qeury laguage.
    • additionally, MongoDB employs BISON documtens that are binary coded into typically smaller files.
    • many developer find BISON easier to manipulate, making data management faster.
  • Simlarities:
    • both of them are database management systems that serve as the ground level information network for any type of digital site or space
    • Both support the same languages like Java or Python.
    • ther're open source therefore the code is freely available fro anyone who wants to use and to distribute.
    • both are a cloud native solution
  • Which to use?
    • MySQL is considered highly accessible and secure, making it well-suited to high traffic sites
      • such as e-commerce sites and compliance heavy industries that require protocols for high security.
    • MongoDB use of dynamic schema design, fosters a more flexible environment for data search, coding, integration and database development.
      • it's optimal for content management systemns and high query sites such as analytics applications.
