Code Assistance

Quick Popup

  • press F1 to see the documentation for the symbol ot the caret.
  • press Esc to close the popup.
  • press Alt-Space to see the definition of the symbol at the caret.

Editor coding assistance

  • press F2 to go to the next highlightd error in the file.
  • fix the error by pressing Alt-CR and select a suggestion
  • also, you can see the warning or error description of the item at the caret. Press Cmd-F1 to see a short description and press Cmd-F1 again to expand it.
  • the popup shows refactoring action suggenstion to fix a warning. Press Alt-Shift-CR to execute the suggenstion.
  • another useful tool is the usages highlighting. Press Shift-Cmd-F7 to highlight all usages of the symbol at the caret within the file.

Navigation

File structure

  • A large source file can be difficult to read and navigate. Sometimes you only need to preview it. Press Cmd-F12 to open the file structure.
  • Suppose you want to find a methid with *Homo* and *Sapiens* words in its name. Type hosa to filter the file structure
  • Only one item remains. Now press Cr to jump to the selecteed item.
  • IntelliJ IDEA can also show you the file structure as a tool window. Open with Cmd-7.

Decleration and usage

  • press Cmd-B to jump to the decleration of a method.
  • Now the caret is at the method's decelatation. Use the same shortcut Cmd-B to see all of its usages, then select one of them.
  • Press Alt-F7 to see a more detailed view of usages. You can invoke Alt-F7 on either a declaration or a usage.
  • From the *Find* view you can navigate both to usages and declarations. The next search will override these results in the Find view. To prevent it, pin the results: right-Click the tab title, foo *In All Places*.
  • Select *Pin Tab*.
    • to close a tab press Ctrl-Shift-F4
  • When you have finished borwsing usages, press Shift-Esc to hide the view.
  • Press Cmd-3 to open the *Find* view again.

Inheritance hierarchy

  • press Alt-Cmd-B to look for implementations of a function of en interface.
  • choose any implementation and press CR or click it.
  • you can navigate to a super method from derived. Press Cmd-U or click the icon in the editor.
  • the decliration of the base method has its own gutter icon. Click it or press Alt-Cmd-B again
  • for big hierarchies, you might want ot look for implementation in the Find tool window. Click on the right icon in the corner.
  • press Shfift-Esc to hide the Finf tool window.
  • you might want to explore the whole hierarchy for this method. Press Shift-Cmd-H.
  • to view class hirarchy, press Ctrl-H -> Conflict with Vim-Mode
  • Note: Actions Alt-Cmd-B and Cmd-U can be also applied to classes. Actkions Shift-Cmd-H and Ctrl-H are rearly used, but you can always find them with Shfit-Cmd-A using the hierarchy filter

Code Completion

Statement completion

  • press Shfit-Cmd-Cr to complete the *for* statement. 
      for (int j = 2; j < i; j++)
    • this will add the brackets to the body
  • type *if* and press Sfhit-Cmd-Cr to generate the statement. 
      if () {
  }
    • Add a condition inside the parentheses *i % j == 0* and press Shift-Cmd-Cr to jump into the *if* statement.
  • type a line of code: *isPrime = false; break* and then press Shift-Cmd-Cr to complete the statement and apply formating
  • the hole code 
    class PrimeNumbers {
    public static void main(String[] args) {
        System.out.println("Prime numbers between 1 and 100");

        for (int i = 2; i < 100; i++) {
            boolean isPrime = true;

            for (int j = 2; j < i; j++) {
                if (i % j == 0) {
                    isPrime = false;
                    break;
                }
            }

            if (isPrime) {
                System.out.print(i + " ");
            }
        }
    }
}

Refactoring

Extract variable

  • press Alt-Cmd-V to extract a local variable from an expression in a code block.
  • If the same expressione occures several time, choose replace all
  • Choose a name for the new varibale or leave it as it is.

Extract method

  • press Alt-Cmd-M to extract a selectd code block into a method.
  • Edit the name of the new method or leave the one that IDE proposed. Then press CR
  • If the code block has severel occurenses, the IDE is asking if you want to replace the next one too.

Refactoring menu

  • IntelliJ IDEA supports a variety of refactoring. Many of htem have their own shortcuts. To access some rarely used refactoring, press Ctrl-T and peview a partial list of them.
  • Suppose you're looking for a refactoring action to replace this expression with a parameter. So we need to choose Introduce Parameter.... Now simply type pa of ip (introduce parameter) to reduce the proposed list.
  • Press CR to start the Introduce Parameter refacotring.
  • To complete refactoring, you need to choose some name or leave the default one and press CR
  • Other usful refactoring actions are:
    • inlive variable -> Alt-Cmd-N
    • indroduce constant -> Alt-Cmd-C
      • opens up a dialog menu to choose the new constant's type, name, parent class and visability.
