In general Git helps orgenize files and future versions of it.

It is a "Version Control" tool. And helps with "Collaboration". This is a system that you can use to keep track of the history of a project as well as allow many people to collaborate on the project together. Difference between Git and GitHup. Git: actual version control software. Created by Linus Torvalds in 2005. Github: web service for projects, that are managed through Git.

-Repository ("Repo"): is an other word for a Project, a repository of files. -Commit: is like a "save", to change a file and make a commit. every commit has a hash -add: putting something into the staging area. Staging are is a place for file you want to commit -Branche: a branch from the main linear lists of commits, the root or "master" -pull request: pull a requested branch into master. Or merge a Branch from a Forked repo into Original. -merge: confirm pull request. -Fork: take an entire other instance or repo from someones public repo. -clone: downloading a repository to your computer. -status: save a change in a git file locally -push -pull -remote: a duplicate instance of a repo that lives somewhere else. In command line your refer to it a origin -init
